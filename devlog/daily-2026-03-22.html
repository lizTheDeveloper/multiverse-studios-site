The night everything broke and the morning everything worked again

Some nights at the studio look like this: three games, four emergency deploys, a crash-on-load, an FPS crisis, a deploy script silently dropping every sprite, and a rate limiter that wasn't limiting anything. By morning, all of it was fixed.

What shipped

MVEE

Four deploys, three of them emergency

The game was crashing on load. Every player saw a black screen stuck on "Initializing..." — a ReferenceError in the game loop that made the simulation unreachable. Emergency deploy v0.1.7 landed the fix.

Then: the Phaser engine was capturing keyboard input globally, which meant players trying to type their password in the Matrix auth modal would trigger in-game actions instead of entering characters. Fixed.

Email-based password reset for Matrix accounts also went live — a missing piece for any player who locked themselves out. And the LLM inference endpoint was diagnosed after norns started returning fallback text instead of real AI responses.

Deploy v0.3.2 bundled the keyboard fix, music gate, and LLM diagnosis together. Four deploys in under twelve hours.

Precursors

From 8 FPS to 30 — and a sprite pipeline that works

The single biggest performance fix since launch. Playtesting revealed the game was running at 8–9 FPS against a 30 FPS target — effectively unplayable. Root cause identified and resolved.

Alongside that: a cascade of rendering fixes. Baby creatures now scale correctly by life stage. Cross-species texture fallbacks no longer silently mask missing assets — they surface as visible errors so they can be found and fixed instead of hiding behind a generic sprite.

The deploy pipeline itself was broken: publish.sh was rsync-excluding all .png files, preventing sprite sheets from reaching production. Every code deploy succeeded. Every image deploy silently failed. Fixed.

Cultures of the Belt

Rate limiter tightened

ARIA's broadcast system had a rate limiter that was still too lenient. Parallel event sources were creating seizure-level screen flashing. Tightened across all six call sites and fifteen event producers.

What we learned

The sprite pipeline break is the instructive failure here. publish.sh excluded all PNGs from rsync. The deploy script ran perfectly for code assets. It just silently dropped every image. Deploy logs showed green. Players saw missing textures.

Deploys need artifact verification, not just exit code checks. A green build that ships no sprites is worse than a red build — at least the red build tells you something is wrong.

What is next

MVEE is stabilizing after a rough patch cycle. The LLM endpoint diagnosis uncovered why norns were falling back to canned responses, and the real fix is in progress.

Precursors at 30 FPS opens up gameplay that was previously invisible to testers. With the rendering pipeline clean and deploy scripts fixed, the creature systems can finally be evaluated at the framerate they were designed for.
